# Create Milestone

## Purpose

Create a repository-local milestone document that can coordinate follow-up issues and specs.

## Required reading

- `README.md`
- `docs/TBPS.md`
- `docs/SPECS.md`
- `docs/WORKFLOWS.md`

## Steps

1. Create a milestone document under `docs/milestones/`.
2. State the goal, scope, non-goals, and expected deliverables.
3. Reference the governing specs, TBPs, and workflow documents.
4. Break the milestone into phases or issue-sized units when useful.
5. Capture exit criteria that can be checked locally.

## Validation

- The milestone can be understood without chat history.
- Cross-references point to repository-local documents.
- Exit criteria are concrete and reviewable.

## Exit criteria

- A concise milestone document exists under `docs/milestones/`.
- Follow-up implementation issues can reference it directly.
