# Start Milestone

## Purpose

Start milestone implementation work with aligned reading, scope control, and a documented plan.

## Required reading

- `README.md`
- `docs/TERMINOLOGY.md`
- `docs/TBPS.md`
- `docs/SPECS.md`
- `docs/TESTING.md`
- The relevant milestone document or issue

## Steps

1. Read the governing docs and the milestone scope.
2. Identify the specs, architecture notes, and TBPs that apply.
3. Confirm what is in scope and what is explicitly out of scope.
4. Prepare a small checklist of the milestone phases.
5. Begin implementation in order, validating each phase before moving on.

## Validation

- The working plan matches milestone scope.
- Required documents are identified before implementation starts.
- Testing expectations are known before changes begin.

## Exit criteria

- Implementation has started with a documented checklist.
- Scope, references, and validation expectations are clear.
