# Add TBP

## Purpose

Add a new Task Best Practice document to the repository.

## Required reading

- `docs/TERMINOLOGY.md`
- `docs/TBPS.md`

## Steps

1. Identify the repeatable work pattern that needs a TBP.
2. Create `docs/tbps/<name>.md` with: purpose, required reading, steps, validation, exit criteria.
3. Add the new TBP to the list in `docs/TBPS.md`.
4. Cross-reference the TBP from relevant milestone or issue templates if appropriate.

## Validation

- TBP document exists and has all required sections.
- `docs/TBPS.md` is updated.

## Exit criteria

- New TBP is listed in `docs/TBPS.md`.
- TBP follows the standard structure.
