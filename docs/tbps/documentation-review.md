# Documentation Review

## Purpose

Review repository documentation for accuracy, authority, and cross-reference quality.

## Required reading

- `README.md`
- `AGENTS.md`
- `docs/TERMINOLOGY.md`
- `docs/TBPS.md`
- `docs/SPECS.md`
- `docs/WORKFLOWS.md`
- `docs/TESTING.md`

## Steps

1. Check that index documents reflect the current repository contents.
2. Verify that authoritative documents do not conflict.
3. Remove ambiguous wording that would force future chat clarification.
4. Confirm new durable terminology is defined.
5. Fix missing or stale cross-references.

## Validation

- Documentation is internally consistent.
- Indexes point to existing files.
- Readers can determine authority and next steps from the repo alone.

## Exit criteria

- Documentation is consistent, concise, and cross-referenced.
- No stale references remain in reviewed areas.
