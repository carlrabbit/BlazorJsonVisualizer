# Create Spec

## Purpose

Create a new authoritative behavior specification for repository-local implementation work.

## Required reading

- `README.md`
- `docs/TERMINOLOGY.md`
- `docs/SPECS.md`
- The related milestone issue

## Steps

1. Confirm the behavior needs an authoritative spec.
2. Choose a concise file name under `docs/specs/`.
3. Define scope, required behavior, constraints, and notable non-goals.
4. Link the new spec from `docs/SPECS.md`.
5. Cross-reference related architecture, decisions, and milestone documents.

## Validation

- The new spec is reachable from `docs/SPECS.md`.
- Terminology matches `docs/TERMINOLOGY.md`.
- The spec states behavior clearly enough to guide implementation.

## Exit criteria

- A concise authoritative spec exists in `docs/specs/`.
- Repository indexes and cross-references are updated.
