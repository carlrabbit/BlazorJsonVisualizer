# Finish Milestone

## Purpose

Close milestone work with repository-local documentation, validation, and checked exit criteria.

## Required reading

- The active milestone document or issue
- `docs/SPECS.md`
- `docs/WORKFLOWS.md`
- `docs/TESTING.md`

## Steps

1. Re-check the milestone goal, deliverables, and exit criteria.
2. Validate the changed files and repository cross-references.
3. Update milestone status or notes if a milestone document exists.
4. Summarize completed scope and any intentionally deferred work.
5. Ensure follow-up work can continue from repository documents alone.

## Validation

- Exit criteria were reviewed against the actual repository state.
- Relevant indexes and references are current.
- Any remaining gaps are explicit rather than implicit.

## Exit criteria

- Milestone deliverables are present and validated.
- Repository-local documents are sufficient for follow-up work.
