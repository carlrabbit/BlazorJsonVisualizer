# Bug Investigation

## Purpose

Investigate a bug with a reproducible scope, evidence, and a repository-local resolution path.

## Required reading

- `README.md`
- `docs/TERMINOLOGY.md`
- `docs/SPECS.md`
- The relevant issue, milestone, and affected specs

## Steps

1. Capture the observed behavior and the expected behavior.
2. Locate the governing spec or note the missing spec gap.
3. Reproduce the bug with the smallest reliable scenario possible.
4. Identify the boundary where behavior diverges from the spec.
5. Apply the minimal fix and update docs if the spec changes.

## Validation

- The bug is reproducible or otherwise evidenced.
- The fix addresses the documented root cause or boundary.
- Fast validation covers the corrected behavior when possible.

## Exit criteria

- The bug is explained and resolved or narrowed to a concrete next step.
- Repository-local docs reflect the intended behavior.
