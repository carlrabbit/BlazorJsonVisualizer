# Feature Implementation

## Purpose

Implement a feature in conformance with repository-local specs and milestone scope.

## Required reading

- `README.md`
- `docs/TERMINOLOGY.md`
- `docs/SPECS.md`
- The relevant milestone and feature spec

## Steps

1. Confirm the feature behavior is covered by a spec.
2. Identify the smallest implementation slice that satisfies the issue.
3. Make focused changes that preserve architectural boundaries.
4. Add or update fast tests when code behavior changes.
5. Update specs or terminology in the same change when needed.

## Validation

- Implementation matches the referenced spec.
- Fast tests cover the changed behavior when code changes exist.
- Documentation stays aligned with the resulting behavior.

## Exit criteria

- The feature is implemented within scope.
- Specs, tests, and terminology are aligned with the change.
